Resume Scanner — System Architecture

PDF → Analysis Pipeline

Trace a resume from upload through every module to final output

Stage 1 — Input
📄
Resume Upload app.py
Streamlit File Uploader · UI Layer
User drags or selects a resume file. Streamlit's st.file_uploader() accepts the file and holds it as an UploadedFile object — essentially a bytes buffer with a filename and MIME type. No file is written to disk.
.pdf .docx UploadedFile object bytes in memory
file.read() → raw bytes
Stage 2 — Parsing · resume_scanner/parser.py
⚙️
ResumeParser parser.py
PDF / DOCX Extraction · Text Cleaning
parse(file_content=bytes, file_type=".pdf") is called. The parser routes to _parse_pdf() using PyMuPDF (fitz) — opens the stream without touching disk, iterates pages, and concatenates raw text. For DOCX, python-docx wraps bytes in io.BytesIO() and reads paragraphs and table cells. Raw text is then passed through _clean_text().
fitz.open(stream=…) io.BytesIO re.sub whitespace strip special chars

📝
Clean Text String
Output of Parser — passed to all 4 analysis modules
A single normalised Python string. Excess whitespace collapsed. Special characters stripped. Consistent newlines. This string is the single source of truth for everything downstream — all four analysis modules receive this same string.

Final Report Streamlit UI
User sees complete analysis · downloadable
All four tab results rendered simultaneously in the browser. User can switch between tabs to explore skills, ATS compatibility, AI probability, and job fit. Optional download button exports a summary report. The entire analysis pipeline ran on a single clean text string produced by the parser — no database, no file writes.
